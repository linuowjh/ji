# 需求文档

## 介绍

「云念」是一个网上扫墓祭奠小程序，旨在为用户提供突破时空限制的线上祭奠服务。该小程序聚焦"庄重、便捷、个性化、长久留存"四大核心需求，既满足传统缅怀情感需求，又通过数字化技术实现纪念内容的永久保存与家族情感传承，避免线下祭扫的地域、时间、环境限制。

## 需求

### 需求 1 - 纪念馆管理

**用户故事：** 作为用户，我希望能够创建和管理逝者的数字纪念馆，以便为逝者建立专属的线上纪念空间。

#### 验收标准

1. 当用户选择创建纪念馆时，系统应允许用户输入逝者的基础信息（姓名、生卒年月、生平简介）
2. 当用户上传逝者照片或视频时，系统应支持多种格式并提供老照片修复功能
3. 当用户选择纪念馆风格时，系统应提供多种模板（中式传统、简约素雅、自然清新等）
4. 当用户定制虚拟墓碑时，系统应提供不同材质和样式的模板选择
5. 当用户输入墓志铭时，系统应支持手写输入生成书法字体或上传手写照片转化

### 需求 2 - 线上祭扫功能

**用户故事：** 作为用户，我希望能够进行各种线上祭扫仪式，以便表达对逝者的缅怀之情。

#### 验收标准

1. 当用户选择献花时，系统应提供多种花卉选择（菊花、康乃馨等）并支持定时送花
2. 当用户点烛时，系统应显示蜡烛点亮动画并支持续烛功能
3. 当用户上香时，系统应提供3柱或9柱选择并模拟香火燃烧效果
4. 当用户供奉供品时，系统应提供虚拟供品选择（水果、糕点等）并支持更新
5. 当用户祈福时，系统应允许输入祈福语并生成电子祈福卡挂在祈福墙

### 需求 3 - 创新纪念形式

**用户故事：** 作为用户，我希望能够通过现代化的方式记录和分享对逝者的思念，以便创造更丰富的纪念体验。

#### 验收标准

1. 当用户录制语音或视频留言时，系统应保存在时光信箱中并支持反复播放
2. 当用户上传纪念照片时，系统应按时间轴整理成生命相册并支持添加文字备注
3. 当用户创建生平故事时，系统应支持文字、图片、视频结合的形式记录人生节点
4. 当用户发起线上追思会时，系统应支持邀请亲友参与并提供实时同步功能
5. 当追思会结束时，系统应生成追思会回放视频

### 需求 4 - 家族联动功能

**用户故事：** 作为用户，我希望能够与分散各地的家族成员共同参与纪念活动，以便实现跨地域的家族情感联结。

#### 验收标准

1. 当用户创建家族圈时，系统应允许邀请亲友加入并设置管理员
2. 当家族成员进行祭扫时，系统应实时同步动态到家族圈
3. 当重要纪念日临近时，系统应提前3天向家族成员推送提醒
4. 当用户查看家族传承档案时，系统应显示家族谱系和家族故事
5. 当家族成员参与集体祭扫时，系统应支持一键参与功能

### 需求 5 - 个人中心管理

**用户故事：** 作为用户，我希望能够便捷地管理我的纪念馆和祭扫记录，同时保护个人隐私。

#### 验收标准

1. 当用户查看我的纪念馆时，系统应集中展示用户创建或参与管理的纪念馆
2. 当用户查看祭扫记录时，系统应按时间线显示所有祭扫行为
3. 当用户设置隐私时，系统应提供公开、私密等选项并支持访客权限设置
4. 当用户需要帮助时，系统应提供使用指南和意见反馈入口
5. 当用户查看访客记录时，系统应显示访客信息和访问时间

### 需求 6 - 后台管理系统

**用户故事：** 作为管理员，我希望能够审核和管理平台内容，以便保障平台合规与服务稳定。

#### 验收标准

1. 当用户创建纪念馆时，系统应审核用户上传的信息和内容
2. 当用户发布留言或祈福语时，系统应过滤敏感词和不当言论
3. 当管理员查看数据时，系统应提供用户数、纪念馆数量等统计信息
4. 当需要备份数据时，系统应支持用户纪念馆数据的备份和导出
5. 当管理功能配置时，系统应支持自定义祭扫节日提醒和模板更新
6. 当开发后台系统时，系统应使用Golang作为后端开发语言

### 需求 7 - 增值服务

**用户故事：** 作为用户，我希望能够购买增值服务来获得更好的纪念体验，以便满足个性化需求。

#### 验收标准

1. 当用户购买高级纪念馆时，系统应提供更大存储空间和定制化功能
2. 当用户需要数据备份时，系统应支持导出为加密文件或云端存档
3. 当用户预订专属追思会时，系统应提供专业策划和技术支持
4. 当用户使用老照片修复时，系统应提供AI修复服务
5. 当用户查看收费标准时，系统应明确显示各项增值服务的价格

### 需求 8 - 合规与安全

**用户故事：** 作为用户，我希望我的个人信息和纪念内容得到安全保护，以便放心使用平台服务。

#### 验收标准

1. 当用户上传内容时，系统应严格审核并禁止违规内容
2. 当收集用户数据时，系统应仅收集必要信息并加密存储
3. 当用户访问纪念馆时，系统应根据隐私设置控制访问权限
4. 当用户情绪低落时，系统应在帮助中心提供心理疏导资源
5. 当处理敏感功能时，系统应避免封建迷信内容并转化为现代化表达
